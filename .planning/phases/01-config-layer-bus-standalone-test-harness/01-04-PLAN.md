<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test-harness page → SDK bus | Debug-panel buttons construct synthetic signal payloads; these must contain geometry/timing only, never field values |
| partner/host → init() | init() is the entry point where an unvalidated config would otherwise reach the bus/selector layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | src/index.js init() config gate | high | mitigate | init() calls validateConfig with no surrounding try/catch, so an invalid config throws and stops initialization before publish/subscribe are exposed (CFG-02 wired end-to-end). |
| T-01-04 | Information Disclosure | harness debug-panel payloads | high | mitigate | Synthetic payloads are built from `getBoundingClientRect()` + `Date.now()` + the element's own `data-heed` attribute only — never `element.value`/`textContent`/cookies/localStorage (No-PII rule, CLAUDE.md). Enforced by the no-PII grep in acceptance criteria. |
| T-01-05 | Information Disclosure | dist/sdk.js bundled config | low | accept | The bundled demo config contains only public CONTRACT.md selector strings and a platformId — no secrets, no PII. Acceptable to inline for a standalone dev harness. |
</threat_model>

| T-01-01 | Tampering | src/config.js validateConfig (V5 Input Validation) | high | mitigate | Hard-fail schema validation (CFG-02): validateConfig throws on any missing/invalid field so the SDK refuses to initialize rather than attaching listeners from an unvalidated/malformed config. No warn-and-continue, no default merge. Enforced by the CFG-02 test block. |
| T-01-03 | Tampering | config/schema.json integrity | low | accept | schema.json is authored by this team, not attacker-controlled input.

# Phase 1 — UI Design Contract

> Visual and interaction contract for the four-screen routed swap-flow skeleton. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase framing (from CONTEXT.md/REQUIREMENTS.md):** This phase's gate is DOM structure and routing correctness — all 7 locked `data-heed` selectors resolving to non-null elements, real Next.js navigation, `?amount=X` round-tripping. "Visual polish beyond realistic UX behavior" is explicitly Out of Scope (REQUIREMENTS.md). This contract is deliberately minimal but still concrete: every value below is a real decision the executor should implement as-is, not a placeholder to bikeshed. Formal dark-theme enforcement is PLAT-03 (Phase 3) — this phase only needs the values below applied consistently as it builds.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (e.g. back arrow icon to "Back" text) |
| sm | 8px | Compact spacing between a row's label and its value |
| md | 16px | Default spacing between stacked rows (fee-row, min-received-row, gas row) |
| lg | 24px | Card/section internal padding; spacing between major screen sections |
| xl | 32px | Gap between the screen heading and the first content block |
| 2xl | 48px | Top padding before the primary CTA block on Screens 3 and 4 |
| 3xl | 64px | Not used this phase (no page-level scroll sections long enough to need it) |

Exceptions:
- **44px minimum touch target** on every interactive element carrying a locked `data-heed` selector (`amount-input`, `proceed-cta`, `confirm-cta`, `back-btn`) and on the Screen 1 "Swap" CTA and Screen 4 "Done" CTA — use shadcn `Button` `size="lg"` (h-11/44px) for all primary CTAs; if `back-btn` renders icon+text, give it `min-h-11` via padding rather than relying on font size alone. This is a real accessibility/tap-reliability requirement for a mobile-only (390px, touch) surface, not decoration.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Usage mapping:
- **Display** — Screen 1 wallet balance figure; the numeric value in Screen 2's `amount-input`.
- **Heading** — Screen-level titles ("Wallet", "Swap", "Confirm Swap", "Swap Complete").
- **Body** — Row primary text (asset names, summary row values on Screen 3, button labels).
- **Label** — Secondary/caption text (row labels like "Fee (0.30%)", "Est. gas fee", helper text under the amount input, asset ticker symbols).

Only these 4 sizes and 2 weights (400, 600) are used anywhere in this phase. No italics, no additional weights.

## Color

Dark theme is locked (PROJECT.md Key Decisions; formal enforcement is PLAT-03/Phase 3, but this phase builds with these values from the start so Phase 3 has nothing to retrofit).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (zinc-950) | Page/body background on all 4 screens |
| Secondary (30%) | `#18181b` (zinc-900), border `#27272a` (zinc-800) | Card surfaces: asset rows (Screen 1), the fee/min-received/gas row group (Screen 2 and 3's summary block) |
| Accent (10%) | `#6366f1` (indigo-500) | **Reserved exclusively for:** Screen 1 "Swap" CTA background, Screen 2 `proceed-cta` background, Screen 3 `confirm-cta` background, Screen 4 "Done" CTA background, and the success checkmark icon fill on Screen 4 (`flow-complete`). Nothing else uses this color — not links, not asset icons, not `back-btn`. |
| Text primary | `#fafafa` (zinc-50) | Headings, primary row values, button labels |
| Text muted | `#a1a1aa` (zinc-400) | Row labels, helper captions, secondary balance line |
| Destructive | `#ef4444` (red-500) | Not used in this phase — no destructive actions exist (no delete/cancel-with-consequence flow). Reserved token only, for future phases. |

`back-btn` (Screen 3) uses the shadcn `Button` `variant="outline"` or `variant="ghost"` on the secondary surface color — explicitly NOT the accent color, to keep accent visually singular to "the action that moves the flow forward."

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Per-screen — see Screen-by-Screen Content Contract below (this phase has 4 distinct primary CTAs, not one) |
| Empty state heading | Not applicable as a list/collection empty state. Screen 2's `amount-input` starts empty before the user types (initial visit) — treated as an input placeholder, not a full empty-state pattern (see below). |
| Empty state body | `amount-input` placeholder text: `"0.00"`. Helper caption under the input: `"Enter an amount to swap"` (shown only when the field is empty). |
| Error state | Not applicable this phase. No client-side validation/error UI is built here — the input accepts any value with no error state (CALC-01..04 real-time validation/calculation is explicitly Phase 2's job per D-01; malformed values only affect display, never computation, in this phase). |
| Destructive confirmation | Not applicable — no destructive actions exist in this phase's scope. `back-btn` is a plain non-destructive navigation action (returns to Screen 2 with the amount preserved, per D-05), not a "discard/cancel" action requiring confirmation copy. |

## Screen-by-Screen Content Contract

Exact copy and static placeholder values, per CONTEXT.md's Claude's-Discretion grants (Screen 1 content, Screen 2 placeholder values). These are static strings for this phase — Phase 2 wires the fee/min-received/gas figures to real computation from the same constants; Phase 1 must not build that logic (D-01).

### Screen 1 — Wallet Overview (`/`)
- Heading: `"Wallet"`
- Balance block (Display size): `"2.4382 ETH"`, secondary line (Label, muted): `"≈ $8,213.44"`
- Asset rows (Card surface, 3 rows): 
  - `ETH` — `"2.4382 ETH"` / `"$8,213.44"`
  - `USDC` — `"1,204.50 USDC"` / `"$1,204.50"`
  - `BTC` — `"0.0421 BTC"` / `"$3,890.12"`
- Primary CTA: `"Swap"` (accent-filled, `size="lg"`, routes to `/swap`) — no `data-heed` attribute required (Screen 1 owns none of the 7 locked selectors).
- Optional icons (`lucide-react`): a generic `Wallet` icon next to the heading is acceptable; do not add per-asset brand icons (scope creep beyond "realistic UX behavior").

### Screen 2 — Amount Entry (`/swap`)
- Heading: `"Swap ETH → USDC"`
- Input label (Label, muted): `"You pay"`
- `data-heed="amount-input"`: `type="number"`, `inputMode="decimal"`, placeholder `"0.00"`, suffix/adornment `"ETH"`
- Helper caption under input (Label, muted, shown only when empty): `"Enter an amount to swap"`. When non-empty, replace with: `"Balance: 2.4382 ETH"`
- `data-heed="fee-row"`: label `"Fee (0.30%)"` (Label), value `"0.0015 ETH"` (Body) — static, per D-01
- `data-heed="min-received-row"`: label `"Min. received"` (Label), value `"1,204.32 USDC"` (Body) — static, per D-01
- Gas estimate row (no locked selector, plain row): label `"Est. gas fee"` (Label), value `"0.0004 ETH (~$1.35)"` (Body) — static, per D-01
- `data-heed="proceed-cta"`: `"Review Swap"` (accent-filled, `size="lg"`, routes to `/swap/confirm?amount=${amount}`)

### Screen 3 — Confirmation (`/swap/confirm`)
- Heading: `"Confirm Swap"`
- Read-only summary (Card surface, per D-04 — full breakdown, not just amount):
  - `"You pay"` — `{amount} ETH` (Body, read from `searchParams`)
  - `data-heed="fee-row"`? — **No.** `fee-row`/`min-received-row` are Screen 2-owned selectors per CONTRACT.md (they don't appear a second time with the same `data-heed` value on Screen 3 — reuse the same static values as plain, unattributed rows: `"Fee (0.30%)"` → `"0.0015 ETH"`, `"Min. received"` → `"1,204.32 USDC"`)
  - `"You receive"` — `"1,204.32 USDC"` (Body, same static value as min-received for consistency)
- `data-heed="confirm-cta"`: `"Confirm Swap"` (accent-filled, `size="lg"`, routes to `/swap/success`)
- `data-heed="back-btn"`: `"← Back"` (outline/ghost variant, `size="lg"`, routes to `/swap?amount=${amount}` — D-05). If using a `lucide-react` `ArrowLeft` icon, keep icon + text (not icon-only) for clarity and to protect the 44px tap target.

### Screen 4 — Success (`/swap/success`)
- `data-heed="flow-complete"` container wraps:
  - A `lucide-react` `CheckCircle2` icon, accent-colored (`#6366f1`), sized ~32px
  - Heading: `"Swap Complete"`
  - Body (muted): `"Your swap has been submitted."`
- Primary CTA: `"Done"` (accent-filled, `size="lg"`, `<Link href="/">` — no query params, satisfying D-06's clean reset)

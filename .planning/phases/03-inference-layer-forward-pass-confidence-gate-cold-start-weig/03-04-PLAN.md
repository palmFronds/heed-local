    performs ONE hand-written backprop step, mathematically equivalent to brain.js's own single
    `trainPattern()` step for a ReLU network, but with ZERO brain.js import anywhere in this file
    (Common Pitfall #4 — importing brain.js anywhere reachable from src/index.js's graph would pull

    Compute the output-layer delta as `deltaOutput[j] = probs[j] - target[j]` for each of the 4 output
    nodes — this is the standard, exact softmax+cross-entropy gradient with respect to the logits
    (matches brain.js's own "error = target − output at the output layer" framing when `probs` is
    treated as that output). Compute the hidden-layer delta as `deltaHidden[i] = (sum over j of
    W2[j][i] * deltaOutput[j]) * reluDerivative(hidden[i])`, where `reluDerivative(h) = h > 0 ? 1 :
    0` — this uses the GENUINE (non-leaky) ReLU derivative, consistent with `forwardPass`'s genuine
    ReLU (INF-02). Do not use a leaky slope here; leaky-relu is a training-time-only trick confined to
    `admin/generate-weights.mjs` (Pitfall #1) and must never appear in this runtime backward pass.
    Update weights WITHOUT mutating the `weights` parameter in place — build and return a NEW object:
    `W2[j][i] -= learningRate * deltaOutput[j] * hidden[i]`, `b2[j] -= learningRate *
    deltaOutput[j]`, `W1[i][k] -= learningRate * deltaHidden[i] * input[k]`, `b1[i] -= learningRate *
    deltaHidden[i]`. Return `{ W1, b1, W2, b2 }` in the same shape `forwardPass` consumes, so the
    output can be assigned straight back to `activeWeights`.
  </action>

    Create `admin/generate-weights.mjs` as a dev-only Node ESM script implementing 03-RESEARCH.md's
    empirically-verified recipe verbatim (D-04): import `brain` from `brain.js` and `fs` from
    `node:fs`; define `canonicalData` as exactly 4 one-hot `{ input, output }` pairs in CLASSES order
    [confusion, price_doubt, trust_gap, flow_friction] encoding touch_hesitation→confusion,
    blur_incomplete→flow_friction, scroll_reversal→price_doubt, back_intent→trust_gap; construct
    `new brain.NeuralNetwork({ inputSize: 4, hiddenLayers: [4], outputSize: 4, activation: 'leaky-relu' })`.
    Use `activation: 'leaky-relu'` NOT `'relu'` — this is the single load-bearing finding of the
    research: plain relu fails 0/15 runs (dead-neuron degeneracy, Pitfall #1) while leaky-relu passes
    15/15 with real margins. This is a training-time-only choice; it does not change the shipped
    forward pass, which still uses genuine ReLU per INF-02. Call `net.train(canonicalData, { learningRate: 0.01, iterations: 20000, errorThresh: 0.005 })`.
    Add function `extractWeights(json)` that returns `{ W1: json.layers[1].weights, b1: json.layers[1].biases, W2: json.layers[2].weights, b2: json.layers[2].biases }` — the explicit reshape
    step required because toJSON()'s `layers[0]` is the empty input layer (Pitfall #3); do NOT write
    the raw toJSON() object. Log the converged iteration count and final error. Write `admin/weights.js`
    via fs.writeFileSync with a header noting it is generated (regenerate with npm run generate-weights)
    and `export default` of the extracted weights object. Then run `npm run generate-weights` so
    admin/weights.js exists as a committed artifact. brain.js is imported ONLY in this .mjs file —
    never from anything reachable via src/index.js.
  </action>

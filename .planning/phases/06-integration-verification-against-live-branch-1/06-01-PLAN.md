<objective>
Stand up the local cross-branch runtime wiring so the already-built Phase 1-5 `sdk.js`
can run against a live Branch 1 (`feat/demo-platform`). This is the "setup" half of D-07's
two-plan split — pure plumbing, no new SIG/INF/RESP/LOG code.

Purpose: Physically make two git branches of the same repo run against each other locally —
a worktree of Branch 1 on `next dev` (:3000), and this branch's receiver (:4310) extended to
serve the SDK bundle and a live-route config — wired together by one uncommitted `<script>`
tag in the worktree. This isolates any wiring/serving failure here from the actual behavioral
verification (Plan 06-02), per D-07's rationale.

Output: An extended `local-receiver/server.js` (two new fixed-path GET routes), a new
`config/demo-platform-live.json` (activeScreens excludes `/`), a rebuilt `dist/sdk.js`, and a
`feat/demo-platform` git worktree whose `app/layout.tsx` has an uncommitted script tag that
loads Heed and calls `Heed.init(liveConfig)` (INTEG-01 setup).
</objective>
